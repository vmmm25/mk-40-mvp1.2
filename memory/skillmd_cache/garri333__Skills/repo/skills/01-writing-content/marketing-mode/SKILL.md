# Marketing Mode — 23 Frameworks

## Framework #1: AIDA (clásico)

```
Atención → Interés → Deseo → Acción

Ejemplo para homepage:
A: "Tu competencia ya usa IA para escalar. ¿Y tú?"
I: "Automatiza el 80% de tus tareas repetitivas en menos de 2 semanas."
D: "Nuestros clientes ahorran 20 horas/semana y aumentan ingresos un 40%."
A: "Empieza gratis hoy → [CTA]"
```

## Framework #2: PAS (Problem-Agitate-Solve)

```
Problema → Agitación → Solución

P: Identificar el dolor exacto del cliente
A: Profundizar en las consecuencias del dolor
S: Presentar tu solución como el alivio

Ejemplo:
P: "¿Pierdes horas buscando información entre miles de documentos?"
A: "Cada hora perdida es dinero. Y tus competidores encuentran lo mismo en segundos."
S: "Con nuestra Knowledge Base IA, cualquier dato aparece en 3 segundos."
```

## Framework #3: Before-After-Bridge

```
Before: La situación actual (dolor)
After: Cómo sería la vida con tu solución
Bridge: Cómo llegar de Before a After

Ejemplo:
Before: "Pasas 3 horas al día respondiendo emails de soporte."
After: "¿Y si tu agente IA lo resolviera el 80% automáticamente?"
Bridge: "Nuestro bot se entrena con tu documentación en 1 día."
```

## Framework #4: FAB (Features-Advantages-Benefits)

```
Feature: Qué hace el producto
Advantage: Por qué es mejor que la alternativa
Benefit: Qué gana el cliente

Feature: "Embeddings vectoriales actualizados en tiempo real"
Advantage: "A diferencia de los chatbots de FAQ estáticos"
Benefit: "Tu equipo siempre responde con información actualizada"
```

## Framework #5: Jobs To Be Done (JTBD)

```
"Contrato este producto para hacer [trabajo] cuando [situación], 
de forma que logro [resultado esperado], sin [obstáculo indeseado]."

Los clientes no compran productos, contratan soluciones para trabajos específicos.

Ejemplo:
JTBD: "Contratan nuestro software cuando necesitan cumplir ISO 27001 
antes de una auditoría, de forma que pasan la certificación sin contratar 
un equipo de compliance, sin estudiar 200 páginas de normativa."
```

## Framework #6: Storytelling de marca

```
Estructura narrativa:
1. Protagonist (cliente ideal, con su problema)
2. Villain (el problema/enemigo)
3. Guide (tu marca, con empatía y autoridad)
4. Plan (tu solución, en 3 pasos simples)
5. Call to Action (claro y directo)
6. Success (cómo se ve el éxito)
7. Failure (qué pasa si no actúan)

El cliente es el héroe. Tu marca es el guía (Gandalf, no Frodo).
```

## Framework #7: Pirámide de Maslow aplicada al marketing

```
Necesidades fisiológicas: seguridad financiera, ahorro
Seguridad: fiabilidad, garantías, reducción de riesgo
Social: comunidad, pertenencia, estatus
Estima: reconocimiento, logros, resultados medibles
Autorrealización: propósito, impacto, legado

Ej. SaaS de seguridad:
Seguridad → "Tu negocio protegido 24/7"
Estima → "Las empresas líderes de tu sector confían en nosotros"
```

## Framework #8: Growth Loops (viral/retention)

```
Acquisition Loop:
Usuario adquirido → Usa el producto → Crea valor → 
Comparte/invita → Nuevos usuarios

Retention Loop:
Usuario activo → Genera datos/historial → 
Producto mejora para él → Más sticky → Mayor retención

Tipos:
• Viral: WhatsApp — necesitas que el otro también tenga la app
• Content: TikTok — UGC que atrae más usuarios
• Sales: LinkedIn — sales rep usa el producto para vender el producto
• Performance: Google Ads — más gasto → más datos → mejores resultados
```

## Framework #9: Pirate Metrics (AARRR)

```
Acquisition:   ¿Cómo encuentra la gente tu producto?
Activation:    ¿Tienen una buena primera experiencia?
Retention:     ¿Vuelven después de la primera vez?
Revenue:       ¿Cómo monetizas?
Referral:      ¿Te recomiendan a otros?

KPIs clave por etapa:
A: CAC, CPC, tráfico orgánico
A: % completaron onboarding, tiempo hasta "aha moment"
R: DAU/MAU, churn rate, cohort retention
R: LTV, ARPU, expansion revenue
R: NPS, viral coefficient, referral rate
```

## Framework #10: Jobs-to-be-Done Competitor Analysis

```
Para cada segmento:
1. ¿Qué trabajo está haciendo el cliente ahora? (solución actual)
2. ¿Cuál es su nivel de satisfacción? (1-10)
3. ¿Qué le frustra de la solución actual?
4. ¿Cuánto pagaría por una solución perfecta?
5. ¿Qué le haría cambiar? (switching trigger)

Regla: Si el switching cost > switching benefit, no cambian.
```

## Framework #11: Hook Pattern para contenido (redes sociales)

```
4 tipos de hooks que funcionan:
1. Número o estadística: "El 73% de los CMOs usa IA mal."
2. Promesa: "Cómo duplicar tus leads sin aumentar presupuesto"
3. Curiosidad: "El error que comete casi todo el mundo con el SEO"
4. Controversia: "El táctico de marketing más sobrevaluado de 2025"

Fórmula de hook para thread/artículo:
[Número | Hook] + [Problema] + [Promesa de solución]:
"5 errores que están matando tu tasa de conversión (y cómo arreglarlos en 1 hora)"
```

## Framework #12: Landing Page CRO Checklist

```
Above the fold:
□ Headline clara: qué haces + para quién
□ Subheadline: beneficio principal o prueba social
□ CTA visible sin scroll
□ Hero image/video relevante
□ No más de 1 CTA por sección

Trust signals:
□ Logos de clientes reconocibles
□ Testimonios con nombre real + foto + cargo
□ Métricas concretas ("Ahorramos 20h/semana a 150+ empresas")
□ Garantía o prueba gratuita
□ Certificaciones/awards relevantes

Structure:
□ Problema → Solución → Beneficios → Social proof → FAQ → CTA
□ Cada sección responde una objeción
□ Mobile-optimized: CTA visible en thumb-zone
□ Page speed < 2s
□ A/B test activo siempre
```

## Framework #13: Email Marketing Sequences

```
Onboarding sequence (5 emails × 7 días):
Day 1: Welcome + quick win (1 acción que da resultado inmediato)
Day 2: El "por qué" de tu solución (story)
Day 3: Feature estrella + caso de uso
Day 5: Caso de éxito de cliente
Day 7: CTA a upgrade/feature premium

Reactivation sequence (3 emails):
Email 1: "Te echamos de menos" + actualización de producto
Email 2: "¿Algo no funcionó bien?" + oferta de ayuda
Email 3: Última oportunidad + descuento/trial

Abandono de carrito:
Email 1 (1h): "¿Olvidaste algo?" + resumen del carrito
Email 2 (24h): Resolver objeciones + testimonios
Email 3 (72h): Urgencia + descuento/bonus
```

## Framework #14: Principios de Cialdini aplicados

```
1. Reciprocidad: Da valor antes de pedir. Free tier, contenido, tool.
2. Compromiso: Small commitments → larger ones. Free trial → paid.
3. Prueba social: "12.000 empresas confían en nosotros"
4. Autoridad: Credenciales, premios, menciones en press
5. Agrado: Personalización, humor, valores compartidos
6. Escasez: "Solo 5 plazas disponibles" (REAL, no fake)
7. Unidad: "Únete a la comunidad de [identidad]"
```

## Framework #15: Posicionamiento (April Dunford)

```
Elementos del posicionamiento:
1. Competitive alternatives: ¿Qué haría el cliente sin ti?
2. Unique capabilities: ¿Qué haces tú que ellos no pueden?
3. Value (not features): ¿Qué valor habilitan esas capacidades?
4. Target characteristics: ¿Para quién es ese valor más importante?
5. Market frame of reference: ¿En qué categoría perteneces?

Ejemplo:
1. Alt: Hojas de Excel + consultor freelance
2. Unique: IA entrenada en +500 normativas europeas
3. Value: Auditoría en horas, no semanas
4. Target: CTOs y CISOs de PYMES que buscan certificación
5. Category: Compliance-as-a-Service automatizado
```

## Framework #16: SEO Content Strategy

```
Content pilar → Cluster model:
Pilar page: "Guía completa de ISO 27001 para empresas" (3000+ palabras)
Clusters: 
  - "¿Cuánto cuesta certificarse ISO 27001?"
  - "ISO 27001 vs SOC 2: diferencias"
  - "Checklist de controles ISO 27001 Annex A"
  - "Cómo prepararse para la auditoría ISO 27001"

Cada cluster linkea al pilar → LinkJuice + topical authority

KWI (Keyword Intent):
Informacional: "qué es ISO 27001" → blog post educativo
Navegacional: "ISO 27001 software" → landing de producto
Transaccional: "ISO 27001 consultoría precio" → landing con CTA
Comparacional: "ISO 27001 vs ISO 9001" → página de comparación
```

## Framework #17: Pricing Psychology

```
Tres opciones (Goldilocks effect):
  Basic:    €49/mes — hace poco → hace que el Pro parezca razonable
  Pro:      €149/mes — el que quieres vender → precio anclado
  Enterprise: Contactar → para deals grandes

Tácticas:
• Precio anual con descuento (20%) → reduce churn, aumenta LTV
• Decoy pricing: el plan medio hace al premium parecer mejor valor
• Precio de la oferta original tachado → €149 ~~€299~~
• Social proof en pricing: "Más popular" en el plan Pro
• Free trial sin tarjeta → mayor conversión, menor LTV inicial
```

## Framework #18: Buyer Personas vs ICPs

```
ICP (Ideal Customer Profile) — descripción de empresa:
• Tamaño: 50-500 empleados
• Industria: Fintech, Legal, Sanidad (datos sensibles)
• Tecnología: Usa AWS/GCP, tiene CTO técnico
• Problema urgente: Auditoría prevista en <6 meses
• Presupuesto: €10k-100k/año

Buyer Persona — descripción de persona:
• Nombre ficticio: "Pablo el CTO"
• Cargo: CTO o CISO
• Motivaciones: No quiere sustos en auditorías
• Miedos: Responsabilidad personal en brecha de datos
• Canales de información: LinkedIn, HN, conferencias ISACA
• Objeciones tipo: "¿Por qué no contratar un consultor?"
```

## Framework #19-23: Growth Tactics

```
#19 Product-Led Growth (PLG):
El producto se vende solo. Freemium → Usage → Conversion.
Clave: definir muy bien el "aha moment" y acelerar llegar a él.

#20 Community-Led Growth:
Construir una comunidad que genera demanda.
Discord/Slack de especialistas → Usuarios activos → Clientes naturales.

#21 Influencer / Creator Marketing:
Micro-influencers (10k-100k) mejor ROI que macro.
Buscar: credibilidad en nicho > follower count.

#22 Retargeting Smart:
Segmentar por comportamiento, no solo por visita.
"Visitó pricing 3+ veces" > "Visitó homepage".
Duración del cookie: 7d para urgencia, 30d para awareness.

#23 NPS-driven growth:
Promotores (9-10) → Pedir testimonios + referrals
Pasivos (7-8) → Pedir feedback + feature request
Detractores (0-6) → Salvar relación antes de churn
```
